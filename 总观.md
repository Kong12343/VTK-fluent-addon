# 端到端 AI 驱动的非结构网格生成与流场预测框架 (v3.0)

## 1. 框架总览

本项目构建一个数据驱动的"神经 CFD 求解器"，从带多维物理标签的边界面端到端输出含流场数值的完整内部计算图。采用**两模块拆分**策略：

**两模块流水线：**

1. **网格生成器：** 从边界几何（坐标、法向、类型）迭代生成内部网格（坐标 + 拓扑）。通过多级精化逐步从少量种子点逼近目标节点数。
2. **场预测器：** 在生成的内部网格上，结合边界物理场，通过有权图 GNN 传播信息，输出内部体元的流场值。

```text
边界几何 (coords, normals, labels)
      │
      ▼
┌─────────────────────────────────────────────┐
│  模块 1: 网格生成器                           │
│  多级迭代精化                                 │
│  种子点 → radius建图 → cross-attention边界注入 │
│  → GNN消息传递 → Δ坐标修正 → 重建图 → 循环      │
│  输出: internal_coords + edge_index           │
└──────────────┬──────────────────────────────┘
               │
               ├─────────────────────────────┐
               │                             │
边界场 (fields) │                       ┌─────▼──────────────────────┐
               │                       │  模块 2: 场预测器             │
               └───────────────────────►  边界编码 → 条件GNN           │
                                       │  带 edge_weight 消息传递     │
                                       │  输出: internal_fields       │
                                       └─────────────────────────────┘
```

模块拆分的好处：几何和物理独立迭代，模块 1 可独立评估网格质量，模块 2 可复用模块 1 的中间结果做消融实验。

## 3. 数据出入口定义

### 部署推理阶段的 I/O (Python 接口)

```python
# 输入：带多维物理边界条件的网格壳
input_data = {
    "boundary_coords": Tensor([N, 3]),     # 边界点坐标
    "boundary_normals": Tensor([N, 3]),    # 边界外法向
    "boundary_labels": Tensor([N, C]),     # One-hot 标签 (Inlet, Outlet, Wall 等)
    "boundary_values": Tensor([N, K])      # 补零对齐后的多维物理约束 (u, v, w, P, T, k, ω...)
}

# 输出：完整的 CFD 计算图与场
output_data = {
    "internal_coords": Tensor([M, 3]),     # 生成的内部节点坐标
    "edge_index": Tensor([2, E]),          # 生成的图拓扑（半径图，正反向各一条边）
    "edge_weight": Tensor([E]),            # 与 edge_index 对齐的边权（面积权重/稀疏化重加权）
    "predicted_field": Tensor([M, K])      # 预测的流场
}
```

### 精度控制接口

```python
# 模块 1 推理接口
internal_coords, edge_index = mesh_generator.generate(
    boundary_geometry,
    target_nodes=5000  # 用户指定目标节点数，自动计算精化级数
)
```

`target_nodes` 越大 → 精化级数越多 → 网格越密。模块内部自动计算需要的精化轮数 L。

## 4. 数据特征工程与多维边界对齐

流体物理量和几何尺度的跨度极大，且不同边界类型的物理量维度不同。本框架采用**"补零门控 + 独立归一化"**机制解决多维特征输入问题。

### 4.1 多维特征张量构造表

| 节点类型 | 变量名 | 维度 | 物理特征描述 |
| :--- | :--- | :--- | :--- |
| **Boundary Face** | X_geo | 3 | 边界型心坐标 (x,y,z) |
| | X_norm | 3 | 边界面法向向量 (n_x, n_y, n_z) |
| | X_label | C | 边界类型独热码，如 `[1,0,0]`(Wall), `[0,1,0]`(Inlet) |
| | X_val | K | 统一扩充为 K 维的多物理量张量 (详见 4.2) |
| **Internal Cell** | X_pos | 3 | 内部控制体质心坐标 (x,y,z) |
| **Topology Edge** | E_attr / w_ij | 4 + 1 | 空间相对位移 (dx, dy, dz) + 欧氏距离 \|\|d\|\|，以及共享面面积权重 A_ij |
| **Target Field** | Y_target | K | 待预测的物理场真值 (如速度、压力) |

### 4.2 异构边界的对齐与门控机制

针对不同边界包含不同物理量（如 Wall 无速度压力输入，Inlet 有速度无压力），执行以下策略：

1. **统一维度 (Zero-Padding):** 将所有边界的物理量强行拼接为固定长度 K 的向量（例如 `[u, v, w, P, T, k, omega]`）。对于该边界缺失的物理量，直接用 `0.0` 填充。
2. **标签门控 (Label Gating):** 神经网络利用输入的独热码 X_label，通过注意力机制自动学习关闭全零无效通道，避免补零对特征提取造成干扰。

### 4.3 独立通道归一化 (Independent Channel Z-Score)

各物理量必须**分别独立**进行标准化。计算均值和方差时，**必须利用 Mask 排除掉补零的假数据**，仅统计有效数值。

### 4.4 内部拓扑前处理：面积加权图

内部计算域表示为面积加权的 cell 邻接图：

```text
G_cell = (V, E, w),   w_ij = A_ij
```

其中每个内部控制体质心是一个节点 v_i ∈ V；两个 cell 共享一个内部面时建立边 (i,j)；边权 w_ij 取共享面的多边形面积 A_ij。正反两条边共享同一面积权重，即 w_{i→j} = w_{j→i} = A_ij。该权重在训练侧作为 `edge_weight` 使用，使 GNN 聚合强度与实际通量交换截面一致。

对超大网格，在训练/推理前可对面积加权图执行 **Benczúr-Karger 割稀疏化**，显著减少边数的同时保留主要流动通道和关键拓扑瓶颈。稀疏化后 `edge_weight` 为重加权值 w'/p，与 `edge_index` 严格对齐。此项为可选优化，首轮以完整面积加权图训练。

## 5. 详细架构实现

### 阶段一：C++ 数据零拷贝提取（已实现）

在 `vtkFLUENTCFFReader` 中解析 Fluent CFF 文件（`.cas.h5`/`.dat.h5`），提取：

* 边界几何：面型心坐标、法向、Zone 类型标签、面面积
* 内部拓扑：体元质心坐标、cell-cell 邻接边、共享面面积权重
* 物理场：边界面与内部体的多维流场数据（取交集对齐）

通过 `FluentCFFGNNExporter` 用 LibTorch 的 `torch::from_blob` 零拷贝组装 Tensor，经 pybind11 传回 Python。已支持字段排除、最大展开列数限制、按 Zone 区间拷贝等优化。

### 阶段二：网格生成器 —— 多级迭代精化

#### 核心思路

不采用固定 3D 晶格模板变形，而是从少量种子点出发，**逐级精化**：每级在上一级粗网格基础上上采样，通过 cross-attention 从边界点注入几何约束，用 GNN 传播信息并微调坐标，最终收敛到目标网格。

坐标微调到位后动态重建 radius graph —— **边由几何决定，无需显式预测边**。

#### 训练

1. 从 CFF 导出的全图 `G_0`（原始网格）用 METIS/Graclus 逐级粗化，构建层级：

```text
G_0 (原始 NC)  →  G_1 (NC/2)  →  G_2 (NC/4)  →  G_3 (NC/8)  → ...
```

1. 训练单级精化模型 `G_k → G_{k-1}`：输入粗网格坐标 + 边界点 embedding，输出上一级精细网格的坐标
1. 随机采样不同层级 k（类似 dropout），让同一模型对所有粗化深度都健壮

#### 单级精化内部流程

```text
输入: pos_k [N_k, 3]（粗网格坐标）
      boundary_embeddings [N_B, d]（边界点经 PointNet++ 编码的逐点特征）

1. 上采样: 每个粗节点附近插值/分裂出 m 个细节点 → pos_seed [N_{k-1}, 3]
2. 建图:   edge_index = radius_graph(pos_seed, r_k)
3. 跨注意: h = CrossAttend(queries=pos_seed, keys=boundary_embeddings)
           ← 每个内部种子点选择性关注相关的边界面区域
4. GNN:    h = GNN(h, edge_index)  ← 邻域消息传递
5. 坐标:   pos_seed = pos_seed + ΔNet(h)  ← 坐标微调
6. 重建:   如果坐标变化超过阈值 → 回到步骤 2 重建图
7. 重复 2-6 直到收敛

输出: pos_{k-1} [N_{k-1}, 3]
```

关键设计点：

* **逐点边界注入**而非全局隐向量 z：每个内部节点通过 cross-attention 选择性关注相关边界面，避免信息瓶颈
* **坐标和图交替精化**：先有坐标才能建图 → 建图后才能传播信息 → 信息更新坐标 → 坐标变化后图也更新
* **Radius graph 替代边预测**：不需要显式的 NeuralEdgePredictor，边合法性由邻近坐标自然保证

#### 推理

```python
def generate(boundary, target_nodes):
    pos = seed_points_from_boundary(boundary)  # 极少量初始种子
    L = compute_refine_levels(pos.size(0), target_nodes, upsample_factor=2)

    for _ in range(L):
        pos = refine_one_level(pos, boundary)  # 同一个训练好的精化模型

    edge_index = radius_graph(pos, r_final)
    return pos, edge_index
```

#### 网格损失函数

```text
L_mesh = α·L_chamfer(pos_pred, pos_gt) + β·L_coord(pos_pred, pos_gt)
```

* **Chamfer 距离**：集合级对齐，允许预测点与 GT 点不严格一一对应
* **坐标逐点 MSE**（当上采样可追溯对应关系时）：更精确但依赖细节点与粗节点的分裂对应

### 阶段三：场预测器 —— 条件有权 GNN

在模块 1 生成的网格骨架上做流场回归：

```text
输入: internal_coords [NC, 3]  (模块 1 输出)
      edge_index [2, E]        (模块 1 输出)
      edge_weight [E]          (保留/重新计算面积权重)
      boundary_fields [NB, K]  (CFF 提取的边界物理场)

流程:
  1. 边界物理场编码: BoundaryEncoder(boundary_geometry + boundary_fields) → z [64]
  2. 条件注入:     x = [coords | z_broadcast]  [NC, 3+64]
  3. 编码器:       h = NodeEncoder(x)  [NC, hidden]
  4. 消息传递:     h = Processor(h, edge_index, edge_weight)  × N_layers
                  ← 每层消费 edge_weight，加权聚合邻居
  5. 解码器:       y_pred = Head(h)  [NC, K]

输出: predicted_field [NC, K]
```

**与当前 `InternalFieldGNN` 的关键区别**：消息传递层必须真正消费 `edge_weight`。当前 `SAGEConv` 不接收 `edge_weight` 参数，需替换为支持加权消息传递的层（如自定义 `WeightedMessagePassing` 或 PyG 中实际支持权重的卷积变体）。

#### 场预测损失函数

```text
L_field = γ·L_MSE(y_pred, y_true)
```

### 阶段四：联合训练（可选端到端精调）

两模块可分别独立训练，也可端到端联合精调：

```text
L_total = α·L_chamfer(coords) + β·L_coord(coords) + γ·L_MSE(fields)
```

首轮建议：模块 1 和模块 2 独立训练，验证各自收敛后再尝试联合微调。

## 6. 精度可调机制

模块 1 通过 `target_nodes` 参数控制输出网格密度：

| target_nodes | 精化级数 L | 预期网格 |
|-------------|-----------|---------|
| NC / 8 | 0 | 极粗（仅种子点） |
| NC / 4 | 1 | 粗 |
| NC / 2 | 2 | 中等 |
| NC | 3 | 细（训练全量） |

* 上采样因子 s ≈ 2（每级大致翻倍节点）
* 节点数取最接近但不超过 `target_nodes` 的 `NC_seed × s^L`
* 同一模型递归调用，无需按精度训练多个模型

## 附录：与 v2.0 的关键变更

| v2.0 | v3.0 | 变更原因 |
|------|------|----------|
| VolumetricFoldingNet（固定 3D 晶格变形） | 多级迭代精化（种子点逐步上采样） | 固定模板无法表达分叉/弯管等复杂拓扑；全局 z 信息瓶颈 |
| NeuralEdgePredictor（显式边预测） | Radius graph 动态重建 | 边是几何的果——邻近坐标自然决定连边，无需单独预测 |
| 全局隐向量 Z 注入 | Cross-attention 逐点边界注入 | 每个内部节点需选择性关注相关边界面，非全局广播 |
| 三模块（FoldingNet+连边器+GNN） | 两模块（生成器+预测器） | 边预测并入生成器的构图逻辑，降低模块耦合 |
| 精度固定 | target_nodes 可调 | 推理时按需选择精度，同一模型递归适用 |
